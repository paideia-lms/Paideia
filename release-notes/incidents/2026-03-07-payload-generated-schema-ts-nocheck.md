# Incident Report: payload-generated-schema Circular Refs and @ts-nocheck

**Date**: March 7, 2026  
**Severity**: Low (typecheck fails in standalone packages)  
**Affected**: `packages/module-user/src/payload-generated-schema.ts`  
**Status**: Resolved  
**Incident ID**: INC-2026-03-07-004

## Summary

When a standalone module package (e.g. module-user) has its own `payload-generated-schema.ts`, TypeScript reports:

```
'users' implicitly has type 'any' because it does not have a type annotation and is referenced directly or indirectly in its own initializer.
'media' implicitly has type 'any' because it does not have a type annotation and is referenced directly or indirectly in one of its return expressions.
Subsequent property declarations must have the same type. Property 'schema' must be of type 'DatabaseSchema', but here has type 'DatabaseSchema'.
```

The schema file is auto-generated by Payload (`payload generate:db-schema`) and contains circular references (e.g. users.avatar → media, media.createdBy → users) plus a `DatabaseSchema` module augmentation that conflicts with `@payloadcms/db-postgres`.

## Impact

**Symptoms**:
- `bun run typecheck` fails with implicit `any` and schema conflicts
- Excluding the file via `tsconfig.exclude` does not help—imports still pull it in for type resolution

**Root Cause**:
- Payload-generated Drizzle schema has circular table references
- The `declare module "@payloadcms/db-postgres"` block re-declares `GeneratedDatabaseSchema.schema` with a type that conflicts

## Resolution

Add `// @ts-nocheck` as the **first line** of the file:

```typescript
// @ts-nocheck - Generated by Payload, has circular refs and schema conflicts
/* eslint-disable */
/**
 * This file was automatically generated by Payload.
 * ...
 */
```

This suppresses all type errors in the file. The file is generated; do not fix the types manually—they will be overwritten on the next `generate:db-schema` run.

## Prevention

For any package that runs `payload generate:db-schema` and produces `payload-generated-schema.ts`:
1. Add `// @ts-nocheck` at the top immediately after generation
2. Consider adding a post-generate script or documenting this in the package README
